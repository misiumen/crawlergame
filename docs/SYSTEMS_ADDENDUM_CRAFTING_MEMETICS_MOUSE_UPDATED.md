# Systems Addendum: Salvage, Improvised Crafting, Memetics, and Mouse UI

This addendum updates the previous implementation plan after the procedural generator review.

## Current known unfinished generator/content issues to respect

Before adding salvage, crafting, memetics, and mouse-only UI, do not ignore the existing open integration work:

1. Wire `risks` / `rewards` metadata into the resolution and consequence layers.
2. Let safehouse `possible_clue_sources` drive clue placement instead of relying only on hardcoded room-type mappings.
3. Make encounter selection tag-aware so generated encounters reinforce the chosen objective, room, crawler, and floor theme.
4. Validate objective `required_tags` against the generated floor before accepting the floor.
5. Add rumor metadata such as `tags`, `weight`, and `floor_min` / `floor_max` so rumors can be generated by tier and context.
6. Treat clue chains as real sequenced unlock chains, not only loose text entries.
7. Connect `item_damage` and item `state["damaged"]` to item degradation and repair.
8. Re-verify save/load after the new `Character.flags["known_clues"]` and `known_facts` keys.

## Design decision

Salvage, crafting, and memetics must be integrated with these systems rather than bypassing them.

Examples:
- Salvage risks should feed the same risk/reward consequence pipeline.
- Crafting can produce damaged/unstable items, and damaged items must matter.
- Memetic belief seeds should create rumors, clues, encounter tags, sponsor reactions, and future consequences.
- Mouse UI should expose known clues, visible interactables, salvage actions, crafting actions, and memetic/social actions only when context supports them.
